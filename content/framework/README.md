# Framework — Development Spine

The **how** of the domain: how a keeper develops over time, how training
is delivered, and how the keeper communicates. The [syllabus](../syllabus/README.md)
is *what* to coach; the framework is *how* to coach it, *when*, and *to
whom*.

| File | What it defines |
|---|---|
| [`development-framework.md`](development-framework.md) | The age/level model — roughly **U9–U12 foundation**, **U13–U15 development**, **U16–U18+ advanced** — with training frequency/duration, technical priorities, and per-stage **testing**. Younger = technique-first; older = technique + positioning + fitness. |
| [`training-methodology.md`](training-methodology.md) | The two delivery methods — **Simple-to-Complex** and **Whole-Part-Whole** (both end in a match-realistic rep) — plus session structure, organization (frequency, field/equipment), and **charting progress** (journal + video). |
| [`communication-syllabus.md`](communication-syllabus.md) | The **four-stage communication ladder** — Stage 1 (own saves) → Stage 2 (individual teammates) → Stage 3 (the back line) → Stage 4 (whole-team tactics) — with the command glossary for each stage. |

## Using the framework

1. Place the keeper on the **development framework** → their stage and emphasis.
2. Build sessions with a **training-methodology** method + the [syllabus](../syllabus/README.md) + [drills](../drills/library.md).
3. Layer in the **communication syllabus** stage-appropriately as the keeper matures.
